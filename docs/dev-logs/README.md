# Dev Logs

Decision-driven engineering journal for `hermetic-agent`. One file per working session, named `<YYYY-MM-DD>-<topic>.md`.

## Convention

- **Lead with the why.** Each entry explains the decision, not just the diff.
- **Include repo ops.** Branch pushes, remote changes, and rebases belong here too — not just code commits.
- **Cite SHAs.** Every claim about "what shipped" links to a commit SHA so future readers can verify.
- **List excluded dirt.** Anything left dirty in the working tree goes in §3 so the next session knows it was a deliberate choice, not an oversight.
