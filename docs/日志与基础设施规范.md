## 2. 日志规范

### 2.1 统一入口：`fh.tools.log.BusiLogger`

模块**不直接使用 SLF4J**。所有业务日志走 `fh.tools.log.BusiLogger`（由 `fh-tools` 提供），由 AOP / 切面 / 节点代码统一注入：

```java
@Autowired
private BusiLogger log;
log.info("%s", "[GraphNode] node=booking_llm outKeys=...");  // 严禁占位拼接
log.error(throwable, "%s", "[ToolCall] tool=cabinChoose status=FAILURE");
```

> 为什么用 `BusiLogger` 而不是 `LoggerFactory`：与 `fh-tools` 日志框架对齐，宿主接入链 C 时无需替换日志门面；AOP 切面（`ToolLogAspect` / `ApiCallLogAspect`）能稳定 grep 关键字面量。

### 2.2 固定前缀（grep 锚点）

**绿场图编排前缀**（`platform/infra/log/LogMarkers.java`）— BusiLogger 正文 grep 锚点，**勿改名**：

| 常量 | 字面量 | 写入方 | 含义 |
|------|--------|--------|------|
| `GRAPH_ROUND_BOUNDS` | `[GraphRoundBounds]` | `RoundBoundsLoggingLifecycle` | 每轮起止 |
| `GRAPH_NODE` | `[GraphNode]` | `GraphNodeTelemetry` | 节点边界 + outKeys |
| `GRAPH_ROUND_SUMMARY` | `[GraphRoundSummary]` | — | 单轮汇总卡片 |
| `ROUND_FLOW_DIGEST` | `[RoundFlowDigest]` | `GraphRoundOrchestrator` | 轮末流程卡（Skill/检索） |
| `GRAPH_TRACE_ANCHOR` | `[GraphTraceAnchor]` | — | 追踪锚点 |
| `GRAPH_SLOW_ROUND` | `[GraphSlowRound]` | — | 慢轮告警 |
| `GRAPH_GOVERNANCE` | `[GraphGovernance]` | `RoundExecutionGovernanceLifecycle` | 治理挂钩失败 |

**观测 API 层前缀**（`platform/infra/log/AgentLogMarkers.java`）— 切面拼装，**勿混用**：

| 常量 | 字面量 | 含义 |
|------|--------|------|
| `API_CALL_MARKER` | `[ApiCall]` | 观测日志正文 |
| `API_CALL_LAYER_PORT` | `[编排入参]` | callSurface=PORT |
| `API_CALL_LAYER_PRODUCT` | `[product调用]` | PRODUCT_SVC / TRAVEL_MANAGER |
| `API_CALL_LAYER_INTEGRATION` | `[integration调用]` | INTEGRATION_SVC |
| `API_CALL_IN_SEGMENT` | `[入参][inParams]` | 入参段 |
| `API_CALL_OUT_SEGMENT` | `[出参][outParams]` | 出参段 |

**Tool 层前缀**（`platform/observe/ToolLogAspect` 拼装，**非 LogMarkers 常量**）：

| 字面量 | 用途 |
|--------|------|
| `[ToolCall]` | 工具调用主行（`requestSeqNo/sessionId/integration/toolName/toolDisplayName/argsSummary/resultSummary/costMs/status/errorMsg/callSeq/toolCallLogId`） |
| `[ToolCallBounds]` | 工具入/出边界对（【工具·开始】/【工具·结束】） |
| `[LlmCall]` | LLM 调用（输入/输出 token） |

### 2.3 MDC 键（`AiRequestContext`）

| MDC key | 形态 | 含义 |
|---------|------|------|
| `main_request_seq_no` | Agent 多轮 `R…D…` / 平台批处理 `P…D…` | 请求流水号 |
| `session_id` | 字符串 | 会话 ID（与 Redis ChatMemory 对齐） |
| `ai_correlation_id` | UUID | 每轮新生 |
| `platform_operation` | 字符串 | 非 Agent 入口逻辑名 |

### 2.4 日志截断与脱敏

由 `ToolLogAspect` 等 AOP 切面强制执行（**业务代码不要重复做**）：

| 字段 | 截断长度 | 出处 |
|------|----------|------|
| `argsSummary`（日志行） | 200 字符 | `ToolLogAspect.SUMMARY_MAX_LEN` |
| 单参数（行内） | 100 字符 | `summarizeArgs` |
| `argsSummary`（DB 持久化） | 4000 字符 | `ai_tool_call_log.argsSummary VARCHAR(4000)` |
| `resultForModel`（DB 持久化） | 65000 字符 | `AiTracePersistenceService` |
| `errorMsg`（行内） | 500 字符 | `ToolLogAspect` |

### 2.5 Banned（日志反模式）

- ❌ **占位符拼接**：`log.info("tool=" + name + " cost=" + cost)` — 改用 `log.info("%s", "tool=" + ...)` 避免 SLF4J 误判占位
- ❌ **改 `[GraphNode]` 等固定字面量**：所有切面/节点/编排器 grep 都依赖这些字面量
- ❌ **`SLF4J LoggerFactory.getLogger`**：统一走 `BusiLogger`
- ❌ **生产常驻 `travel-ai-debug` profile**：会刷屏 `[GraphRoundBounds]` / `[ToolCallBounds]` / 上下文审计指纹
- ❌ **自己写 `KEY_LAST_TOOL_*` / `KEY_TOOLS_CALLED_ORDER`**：仅允许 `AiRequestContext.round().put(...)`，由 `ToolLogAspect` 簿记

## 3. Redis 接入（生产**强制**）

### 3.1 关键约束

- **生产固定 Redis** — 没有 in-memory 模式
- **必须显式声明**：`spring.data.redis.url` 或 `spring.data.redis.host`（或兼容的 `spring.redis.host`）— `RedisInfraRequirements.requireExplicitConnection` 在启动 fail-fast
- **没有默认值** — 启动即抛 `IllegalStateException("未在宿主配置中显式声明 Redis 连接...")`
- `RedisChatMemoryRepository` 与图 `Checkpoint` 共用 `StringRedisTemplate`（**不**用 Redisson / Lettuce 之外方案）

### 3.2 唯一根前缀

```
ai:           # 业务 Redis 唯一根前缀（见 RedisKeyManager.APP_SEGMENT）
```

> **禁止**在返回值上**再拼** `InfraConstants.LOGICAL_NAMESPACE = "ai"`，否则会出现 `ai:ai:…`（本类键已含 `ai:`）。

### 3.3 键命名（运维 SCAN 锚点）

| 键形态 | 用途 | 写入方 |
|--------|------|--------|
| `ai:ctx:v1:domain:<segment>:<sessionId>` | 领域快照 JSON | `RedisContextStore` |
| `ai:chatmem:v1:msg:<conversationId>` | ChatMemory 消息 | `RedisChatMemoryRepository` |
| `ai:chatmem:v1:ids` | 会话 id 索引 Set | `RedisChatMemoryRepository` |
| `ai:chatmem:v1:last-domain:<sessionId>` | 上一轮路由域 wire | `GraphChatFacade` |
| `ai:timeline:draft:<mainRequestSeqNo>` | 执行时间线草稿 | `ExecTimelineLogRecorder` |
| `ai:reasoning:draft:<sessionId>:<mainRequestSeqNo>` | 思维链草稿 | `RoundReasoningAccumulator` |
| `Ai:Weather:<md5SubKey>` | 天气缓存（链路 C 共用） | 天气服务 |
| `ai:graph:v1:checkpoint:<threadId>` | 图 checkpoint JSON | `StringRedisTemplateCheckpointSaver` |
| `ai:graph:v1:thread:<threadName>` | 图线程元数据 Hash | `StringRedisTemplateCheckpointSaver` |
| `ai:graph:v1:lock:<threadName>` | 图 checkpoint 分布式锁 | `StringRedisTemplateCheckpointSaver` |

`<segment>` 取值（`RedisKeyManager.SEGMENT_*`）：

| segment 常量 | 值 | 业务 |
|--------------|----|------|
| `SEGMENT_AIR_DOMESTIC_BOOKING` | `air_domestic_booking` | 国内订座 |
| `SEGMENT_AIR_INTERNATIONAL_BOOKING` | `air_international_booking` | 国际订座 |
| `SEGMENT_AIR_DOMESTIC_REFUND` | `air_domestic_refund` | 国内退改 |
| `SEGMENT_AIR_INTERNATIONAL_REFUND` | `air_international_refund` | 国际退改 |

通配：`ai:ctx:v1:domain:air_*`（按产品）、`ai:graph:v1:*`（图子树），**慎用 KEYS，建议 SCAN**。

### 3.4 最小接入配置（宿主 YAML）

```yaml
spring:
  data:
    redis:
      host: redis-host.internal       # 必填其一：url 或 host
      # url: redis://redis-host.internal:6379/0   # 二选一
      port: 6379
      password: ${REDIS_PASSWORD:}
      timeout: 2s
      lettuce:
        pool:
          enabled: true
          max-active: 16
          max-idle: 8
          min-idle: 2
```

### 3.5 编译期常量（**勿改 yml**）

```java
// InfraConstants.java
public static final String LOGICAL_NAMESPACE = "ai";
public static final String REDIS_STORE = "redis";
public static final int CHAT_MEMORY_MAX_MESSAGES = 80;       // 滑动窗口条数
public static final int CHAT_MEMORY_TTL_SECONDS = 0;         // 0=不过期
```

### 3.6 启动校验链

`MandatoryInfraGuard`（`@ConditionalOnBean(DataSource.class)`）在 `ApplicationReadyEvent` 之后 fail-fast：

1. `RedisInfraRequirements.requireExplicitConnection(env)` — 至少存在一个 `spring.data.redis.url` / `spring.data.redis.host` / `spring.redis.host`
2. `requireStringRedisTemplateBean(ctx)` — `StringRedisTemplate` 存在
3. `requireGraphCheckpointSaver(ctx)` — 必须是 `StringRedisTemplateCheckpointSaver`
4. `requireRedisChatMemory(ctx)` — 必须是 `RedisChatMemoryRepository`，且只有唯一一个 `ChatMemory` Bean

> 任意一步失败 → `IllegalStateException` → 启动失败。**不要**屏蔽该守卫。

### 3.7 ChatMemory 写入

唯一写入入口是 `ChatMemoryBridge` → Spring AI `ChatMemory`，**业务代码禁止**直接 `ChatMemoryRepository` 写：

```java
// ✅ 正确
chatMemoryBridge.add(sessionId, userMessage);
chatMemoryBridge.add(sessionId, assistantMessage);

// ❌ 禁止
redisChatMemoryRepository.saveAll(sessionId, messages);  // 绕过窗口/记账
```

## 4. MySQL 接入

### 4.1 Schema 总览（11 + 9 = 20 张表，本模块专用 `ai_*` 前缀）

> 详细 DDL：`src/main/resources/db/migration/` — 见 `db/migration/README.md` 完整清单

**A. 追踪域 9 表**（运行时**是**真源，DB 写入主路径）：

| 表 | Java Entity | 主键 | 备注 |
|----|-------------|------|------|
| `ai_session` | `AiSession` | `sessionId` | 无 Long id，DAO 原生 upsert |
| `ai_request` | `AiRequest` | `id` | `SeqNoUtils` / `AiTraceLongId` |
| `ai_chat_message` | `AiChatMessage` | `id` | `recordId` UNIQUE；问法存 `contentJson` |
| `ai_llm_call_log` | `AiLlmCallLog` | `id` | `(requestId, callSequence)` UNIQUE |
| `ai_llm_hop_log` | `AiLlmHopLog` | `id` | function-calling 内层 hop |
| `ai_tool_call_log` | `AiToolCallLog` | `id` | `(requestId, callSequence)` UNIQUE |
| `ai_service_call_log` | `AiServiceCallLog` | `id` | 挂 `toolCallId` |
| `ai_agent_config` | `AiAgentConfig` | `id` | 启动注册 MODULE/TOOL |
| `ai_trace_record` | `AiTraceRecord` | `id` | 编排骨架流水；`requestId` 可空 |

**B. 路由域 11 表**（运行时**否**真源；表驱动预留，Java `DefaultRoutingContribution` 仍是真源）：

- `ai_ref_*` × 2 + `ai_routing_*` × 9
- wire 目录四域：`DOMESTIC_BOOKING` / `INTERNATIONAL_BOOKING` / `DOMESTIC_REFUND` / `INTERNATIONAL_REFUND`
- 兼容别名（解析层）：`REFUND` → `DOMESTIC_REFUND`；`BOOKING` → `DOMESTIC_BOOKING`
- **已下线**：`TRAIN_BOOKING`（运行时回落 `DOMESTIC_BOOKING`）

**C. 安全约定**：所有 DDL 脚本**仅操作 `ai_*` 前缀表**，不会影响同库其他业务模块的表。

### 4.2 首次部署

```bash
# 来自 src/main/resources/db/migration/README.md
mysql -u<user> -p < create_ai_routing_config_tables.sql
mysql -u<user> -p < create_ai_trace_tables.sql
```

字符集与引擎：**MySQL 8.0+ / InnoDB / `utf8mb4_unicode_ci`**，列名 **camelCase**（与 `entity/*` 及 `fh-tools` `BaseBeanDAO` 反射一致）。

### 4.3 最小接入配置（宿主 YAML）

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql-host.internal:3306/fh_travel?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: ${DB_USER}
    password: ${DB_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      pool-name: fh-travel-ai-hikari
      maximum-pool-size: 20
      minimum-idle: 4
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000

mybatis:
  mapper-locations: classpath*:mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true
    default-fetch-size: 100
```

### 4.4 DAO 写入

业务 / 观测代码通过 `platform/observe/persistence/AiTracePersistenceService` 间接落库：

```java
@Autowired
private AiTracePersistenceService trace;

Long toolLogRowId = trace.startToolCallRow(toolName, callSeq, moduleId);
trace.completeToolCallRow(toolLogRowId, argsSummary, resultForLlm, costMs, status, errorMsg);
```

> **不要**直接注入 `*DAO` 写追踪表 — 切面/节点会与你的写入并发冲突，破坏 `callSequence` UNIQUE 与外键语义。

### 4.5 条件装配

- `MandatoryInfraGuard` 用 `@ConditionalOnBean(DataSource.class)` — 单元测试/纯图执行环境下不会触发 Redis 强校验
- 单元测试需要 DB 时**必须**用 `@TestPropertySource` 隔离；不能动默认 `test` task

### 4.6 重建（开发期）

```bash
mysql -u<user> -p < drop_all_ai_dev.sql             # 仅 ai_* 表，不动其他业务表
mysql -u<user> -p < create_ai_routing_config_tables.sql
mysql -u<user> -p < create_ai_trace_tables.sql
```

> `alter_ai_trace_widen_columns.sql` 是已建旧库加宽列用，**新库不要执行**。

## 6. 一页式接入清单（PR 自检）

修改 / 新增任何 **infra / 观测 / 持久化** 代码时：

- [ ] 日志统一走 `BusiLogger`，**不**用 `LoggerFactory`
- [ ] 固定前缀（`[GraphNode]` / `[ToolCall]` / `[ApiCall]` 等）**只引用** `LogMarkers` / `AgentLogMarkers` 常量，**不**自造字面量
- [ ] 长字符串 / 入参出参走 AOP 切面截断（200 / 4000 / 65000）
- [ ] 写 Redis：键名**只**通过 `RedisKeyManager` 方法，**不**手拼 `ai:`
- [ ] 写 MySQL 追踪表：只通过 `AiTracePersistenceService`，**不**直接调 `*DAO`
- [ ] 写 ChatMemory：只通过 `ChatMemoryBridge`，**不**直接调 `RedisChatMemoryRepository`
- [ ] 修改 `*GraphConfiguration` ⇒ 同步 `docs/NODE_STAGE_TABLE.md` + 跑 `verifyNodeStageTable`
- [ ] 默认 `test` 通过（不依赖外部 MySQL / Redis / DashScope）；需要外部依赖的测试用 `@TestPropertySource` 隔离
- [ ] 新增/修改配置项确认属于 A / B / C 哪类真源
- [ ] 启动校验链（`MandatoryInfraGuard`）不能屏蔽
