### 关键设计

1. **不破坏向后兼容**: 旧模块级常量 (e.g. `OPENCODE_ADMIN_PORT`, `BLOCKED_PATTERNS`, `EngineLauncher.FORBIDDEN_CWDS`, `DEFAULT_SSE_KEEPALIVE_INTERVAL`) 全部保留作为**兜底**; 真正校验走 `get_settings()` 读取的实时值; 老 `from ... import` 不报错.

### 4 种启动组合 (一个 compose 搞定, 单 service + PULL_POLICY env)

| 模式 | 命令 | 镜像来源 |
|---|---|---|
| `local` (默认) | `docker compose up -d --build` | 本地 Dockerfile 构建 (`:dev` tag) |
| `registry` (生产) | `PULL_POLICY=always docker compose up -d` | registry pull (用 `OPENAGENT_HUB_IMAGE` 等) |
| `local` + frontend | `docker compose --profile frontend up -d --build` | 本地 + frontend |
| `registry` + frontend | `PULL_POLICY=always docker compose --profile frontend up -d` | registry + frontend |

### 关键设计: 单一 compose + 单 service + `pull_policy: missing`

每个 service 同时声明 `image:` + `build:` + `pull_policy`:

```yaml
openagent-hub:
  image: ${OPENAGENT_HUB_IMAGE:-openagent:dev}    # image 模式用
  build:                                            # local 模式用
    context: .
    dockerfile: docker/Dockerfile.openagent
  pull_policy: ${PULL_POLICY:-missing}             # 跨版本兼容 (spec 核心值, 非 BuildKit extension)
```

- `missing` (默认): 本地有就用, 没才拉 — 等价"优先 build 产物", 跟原 `pull_policy: build` 行为一致
- `always` (PULL_POLICY=always): 强制从 registry 拉, 忽略本地
- 不依赖 `pull_policy: build` 这个 BuildKit extension (跨 compose 版本兼容, 包括 Windows Docker Desktop)
- 模式名 `local` / `registry` 跟 `docker compose build` / `pull` 命令零冲突
- 默认 `docker compose up -d` 直接工作, 无需传 profile (无 "no service selected" 问题)
