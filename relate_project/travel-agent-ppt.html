05 · TOOL CONTRACT

Tool Contract: 协议化强约束

让 Agent "按规矩办事"

  • 前置条件: 必须先选航班才能选舱位
  • 互斥状态: 已选 A 航班时不能再选 B
  • 参数约束: 日期格式、舱位枚举、价格区间
  • 失败回滚: 价格变动 / 政策超规 主动 ask
  • 边界控制: "生成预览,不直接提交" 由工具实现

业务规则 下沉到工具层 · LLM 只负责意图理解与编排
减少幻觉 · 可测试 · 可审计

09 · WHY US

我们的 核心优势

① 协议化,不是 Prompt 堆砌

业务规则下沉到 Tool Contract,LLM 只负责意图理解。规则可测试、可审计、可版本化,不是"看 Prompt 脸色"。

② Runtime 不动,Skill 即插拔

新增酒店/行程/差标等能力 = 新增 Skill,不改 Agent Core。能力扩展风险与成本大幅降低。

③ 后端是事实唯一来源

所有数据走 MCP 调用飞鹤 BSS,LLM 不臆造航班/价格/舱位。差旅是高合规场景,这是底线。

④ 强状态 · 可恢复

Tool Contract 内置前置条件/互斥/回滚;异常可解释、可续办;价格变动/政策超规主动 ask 用户。

⑤ 卡片驱动 · 体验原生

AGUI 协议让前端只按 card_type 渲染。航班/舱位/订单/审批卡片,Web、企微、飞书、App 复用同一份 Card Schema。

⑥ 已落地,不是 PPT

V1.0 国内机票全链路已 ship,38+ 真实演示问题可跑通。范式已被业务验证,后续扩展是同一个故事的延续。
